Вот здесь точно ничего интересного. Сюда пихаю разные технические проблемы и решения, связанные в ведением данного блога. Где-то это надо сохранять и в первом посту самое место.


* Устанавливать gem github-pages надо только для того, чтобы получить точную
  копию окружения, работающего на github. Для сложного сайта это обязательно,
  для меня, скорее всего,  не нужно. Устанавливает нативные расширения, еще
  и сырые - требует полной цепочки autoconf и разные библиотеки. Все это не требуется, если установить просто `gem install jekyll`.

* Однако, вот такая проблема. Для руссификации и подгонки под себя скопировал
  все содержимое джема Minima к себе, однако github отказался ее компилировать.
  Заменил на более раннюю версию, которая установилась с github-pages, все
  стало нормально.

* Лучше всего создавать пустой репозиторий LOGIN.github.io, и затем копировать
  в него содержимое папки, полученной по команде `jekyll new PROJECT`. Тема
  mininal, которую предлагает github предназначена для простого сайта - не для
  блога.

* Как русифицировать даты так и не сообразил. Ну и не надо. Если это станет
  важно, то надо просто установить мультиязычную тему. Minimal mistakes вроде
  хороша.
